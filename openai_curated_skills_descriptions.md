# OpenAI Curated Skills Descriptions

Source: OpenAI curated skills list and each skill's `SKILL.md` description from the `openai/skills` repository.

## Skills

1. **aspnet-core** — Build, review, refactor, or architect ASP.NET Core web applications using current official guidance for .NET web development.
2. **chatgpt-apps** — Build, scaffold, refactor, and troubleshoot ChatGPT Apps SDK applications that combine an MCP server and widget UI.
3. **cloudflare-deploy** — Deploy applications and static sites to Cloudflare using the recommended Cloudflare deployment workflow.
4. **develop-web-game** — Build or iterate on a web game with a reliable development and Playwright-based testing loop.
5. **doc** — Read, create, or edit `.docx` documents, especially when formatting or layout fidelity matters.
6. **figma** — Use the Figma MCP server to fetch design context, screenshots, variables, and assets from Figma, and translate Figma nodes into production code.
7. **figma-implement-design** — Translate Figma nodes into production-ready code with 1:1 visual fidelity using the Figma MCP workflow.
8. **frontend-skill** — [Description __not_ available from official documentation]
9. **gh-address-comments** — Help address review or issue comments on the open GitHub PR for the current branch using the GitHub CLI.
10. **gh-fix-ci** — Debug or fix failing GitHub PR checks that run in GitHub Actions by inspecting checks and logs, then proposing or implementing fixes.
11. **imagegen** — Generate or edit images via the OpenAI Image API, including inpainting, masking, background replacement, transparent backgrounds, and batch variants.
12. **jupyter-notebook** — Create, scaffold, or edit Jupyter notebooks for experiments, explorations, or tutorials.
13. **linear** — Manage issues, projects, and team workflows in Linear.
14. **netlify-deploy** — Deploy web projects to Netlify using the Netlify CLI.
15. **notion-knowledge-capture** — Capture conversations and decisions into structured Notion pages such as wiki entries, how-tos, decisions, or FAQs.
16. **notion-meeting-intelligence** — Prepare meeting materials with Notion context and research, including agendas, pre-reads, and attendee-tailored materials.
17. **notion-research-documentation** — Research across Notion and synthesize findings into structured documentation such as briefs, comparisons, or reports with citations.
18. **notion-spec-to-implementation** — Turn Notion specs into implementation plans, tasks, and progress tracking.
19. **openai-docs** — Use current official OpenAI documentation to answer product or API questions, choose models, and guide upgrades.
20. **pdf** — Read, create, or review PDF files where rendering and layout matter, with visual checks and Python-based extraction or generation.
21. **playwright** — Automate a real browser from the terminal for navigation, form filling, screenshots, extraction, and UI-flow debugging.
22. **playwright-interactive** — Use a persistent Playwright session for fast iterative UI debugging in web or Electron apps.
23. **render-deploy** — Deploy applications to Render by analyzing codebases, generating `render.yaml` Blueprints, and providing deployment links.
24. **screenshot** — Capture desktop or system screenshots when the user explicitly asks for a screenshot or when an OS-level capture is needed.
25. **security-best-practices** — Perform language- and framework-specific security best-practice reviews and suggest secure-by-default improvements.
26. **security-ownership-map** — Analyze git repositories to build a security ownership topology, compute bus factor and sensitive-code ownership, and export graph data.
27. **security-threat-model** — Produce repository-grounded threat models that enumerate trust boundaries, assets, attacker capabilities, abuse paths, and mitigations.
28. **sentry** — Inspect Sentry issues or events, summarize recent production errors, and pull basic Sentry health data via the Sentry API.
29. **slides** — Create and edit PowerPoint slide decks (`.pptx`) with PptxGenJS, layout helpers, and rendering or validation utilities.
30. **sora** — Generate, edit, extend, manage, download, or batch-plan Sora videos, including reusable non-human character references.
31. **speech** — Generate text-to-speech narration, voiceovers, accessibility reads, audio prompts, or batch speech outputs via the OpenAI Audio API.
32. **spreadsheet** — Create, edit, analyze, or format spreadsheets (`.xlsx`, `.csv`, `.tsv`) with formula-aware workflows and visual review.
33. **transcribe** — Transcribe audio files to text with optional diarization and known-speaker hints.
34. **vercel-deploy** — Deploy applications and websites to Vercel, defaulting to preview deployments unless production is explicitly requested.
35. **winui-app** — Bootstrap, develop, and design modern WinUI 3 desktop applications with C# and the Windows App SDK.
36. **yeet** — Stage, commit, push, and open a GitHub pull request in one flow using the GitHub CLI.
